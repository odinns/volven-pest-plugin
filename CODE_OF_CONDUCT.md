# Code of Conduct

Be direct. Be useful. Do not be a nuisance.

Expected behavior:

- critique the work, not the person
- keep reports concrete and reproducible
- assume maintainers have limited time
- respect boundaries when security or private context is involved

Unacceptable behavior:

- harassment, threats, or personal attacks
- public vulnerability disclosure before a fix is coordinated
- spam, entitlement, or issue noise dressed as urgency

Maintainers may close issues, remove comments, or block contributors who make the project harder to maintain.
